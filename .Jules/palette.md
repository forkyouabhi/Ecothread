## 2024-04-08 - Semantic HTML for Interactive Elements
**Learning:** Found instances where `<small>` elements were being used as buttons (with `.btn` classes and JavaScript click handlers). This completely breaks keyboard accessibility because `<small>` tags are not naturally focusable, so keyboard-only users cannot tab to them or activate them. Furthermore, screen readers do not announce them as interactive controls.
**Action:** Always replace non-interactive elements (like `div`, `span`, `small`, `a` without `href`) acting as buttons with native `<button type="button">` elements. This provides built-in keyboard support (focus and Space/Enter key activation) and appropriate screen reader semantics. Include descriptive `aria-label` attributes when the button text alone (like "Update" or "Delete") lacks sufficient context.
